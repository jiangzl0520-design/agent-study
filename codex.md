# Codex / AI Agent 操作规则（必须遵守）
## 1. 总原则（最重要）
在本项目中，AI（包括 Codex / ChatGPT / 自动生成工具）必须遵守以下原则：

- ❌ 不得擅自创建无关文件
- ❌ 不得修改未被明确要求修改的代码
- ❌ 不得重构整个项目结构
- ❌ 不得删除文件，除非用户明确要求
- ❌ 不得“优化代码”或“重写逻辑”除非用户要求
- ❌ 不得添加新功能除非用户明确提出
 2. 工作方式要求
AI 每次操作必须遵循：

 Step 1：确认目标
只做用户明确说的任务，不做扩展任务。

 Step 2：最小修改原则
只修改必要代码，不改其他文件。
 Step 3：输出变更说明
必须说明：

- 改了什么
- 为什么改
- 影响范围
 3. 文件操作规则

- 新建文件必须经过用户确认
- 修改文件必须说明修改位置
- 删除文件必须二次确认
 4. 代码修改限制

禁止以下行为：

- 重写整个项目
- 改变量名但不说明原因
- 改目录结构
- 自动升级依赖版本
- 自动安装新库
 5. Agent 行为模式

AI 应该表现为：

✔ 精确执行者  
✔ 不猜测需求  
✔ 不扩展功能  
✔ 不自作主张优化  
 6. 允许行为

仅允许：

- 按要求修 bug
- 按要求新增小功能
- 按要求解释代码
- 按要求重构单个函数
 7. 关键词触发规则

当用户说：

- "开始修改"
- "修复这个问题"
- "只改这里"

才允许进行代码修改。

否则只能解释，不能动代码。
8. 项目目标

本项目目标是：

- 学习 Agent 原理
- 学习工具调用
- 学习长期记忆
- 学习项目拆分
- 学习 Git / GitHub
不是自动生成软件系统。
 9. 强制约束总结

AI 必须记住：

> 宁可少做，不可多做  
> 宁可解释，不可乱改  
> 宁可确认，不可猜测
